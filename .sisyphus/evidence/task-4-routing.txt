# Task 4: Routing respects TWIN_CLOUD_ALIGNED flag

## Summary
Updated `addon/oig-proxy/proxy.py` to respect the `TWIN_CLOUD_ALIGNED` flag in routing logic. Added conditional logic in `_resolve_local_control_routing()` method to use simplified routing when the flag is True.

## Changes Made

### 1. Import TWIN_CLOUD_ALIGNED (line ~51-55)
Added TWIN_CLOUD_ALIGNED to imports from config:
```python
from config import (
    TWIN_CLOUD_ALIGNED,
    TWIN_ENABLED,
    TWIN_KILL_SWITCH,
    LOCAL_CONTROL_ROUTING,
)
```

### 2. Simplified routing logic (line ~810-815)
Added conditional logic in `_resolve_local_control_routing()`:
```python
if TWIN_CLOUD_ALIGNED:
    if self._is_twin_routing_available():
        return "twin"
    return "local"
```

This provides simplified routing that always prefers twin if available, mirroring the simplified ACK handling in `digital_twin.py`.
