Cross-Session Failure Pattern
-----------------------------
Both failures share identical structure:
  1. Setting delivered mid-session on conn N
  2. BOX closes conn N normally after ~24s (before ACK timeout)
  3. BOX reconnects as conn N+1
  4. ACK timeout fires ~34s after delivery (while BOX is on conn N+1)
  5. Setting transitions DELIVERED → FAILED with no retry mechanism

This confirms the core setting-ack parity bug: the proxy's ACK timeout
is scoped to delivery time, not connection lifetime, causing cross-session
failures when BOX reconnects before timeout expiry.
