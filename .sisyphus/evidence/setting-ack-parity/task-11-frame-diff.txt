### Key Findings

#### 1. Fundamental Architecture Difference (HIGH SEVERITY)
- **Cloud**: Setting delivery observed indirectly via `tbl_events` DATA frame
- **Mock**: Explicit SETTING frame with full XML structure

#### 2. Cross-Session Timeout Pattern (HIGH SEVERITY)
- **Mock**: ACK timeout fires on DIFFERENT connection than delivery (conn_N -> conn_N+1)
- **Cloud**: Protocol ACK within same TCP session (<1s)

#### 3. Connection Lifecycle Difference (HIGH SEVERITY)
- **Mock**: Delivery connection closes before timeout fires
- **Cloud**: Single connection handles complete exchange

#### 4. ACK Timing (HIGH SEVERITY)
- **Cloud**: Protocol ACK is sub-second (within IsNewSet TCP session)
- **Mock**: Explicit ACK timeout at ~30-35s (config-based threshold)

#### 5. Setting Frame Fields (MEDIUM SEVERITY)
Mock Setting frames contain these fields not captured in cloud baseline:
- ID (frame ID)
- ID_Device
- ID_Set
- ID_SubD
- DT (datetime in local format)
- NewValue
- Confirm
- TblName
- TblItem
- ID_Server
- mytimediff
- Reason
- TSec (timestamp in UTC format)
- ver (firmware version)
- CRC

#### 6. Duplicate Tags Note (LOW SEVERITY)
- Spec notes `Reason` tag can appear multiple times
- Current mock frames show single Reason
- Future cases may have duplicates
