=== Dedup Analysis ===

The ForensicOutboundLogger design ensures no duplicate forensic writes:

1. MONOTONIC SEQUENCE COUNTER
   - Each ForensicOutboundLogger instance maintains a monotonic `_seq` counter
   - Starts at 0, incremented BEFORE each record is written
   - Guarantees unique seq per server lifetime
   - Combined with conn_id, creates a unique (conn_id, seq) tuple per record

2. SINGLE WRITE POINT PER OUTBOUND FRAME
   - Each `writer.write()` call has exactly ONE corresponding `_log_outbound_frame()` call
   - The forensic log call is placed BEFORE the wire write, not after
   - No conditional branches lead to double-logging of the same frame
   
3. WRITE POINT AUDIT (4 total):
   - Point 1 (error_banned):     Single execution path, breaks after write
   - Point 2 (end_after_ack):    Single execution path, setting_ack_handled=True prevents re-entry
   - Point 3 (end_after_ack_legacy): Single execution path, setting_ack_handled=True prevents re-entry  
   - Point 4 (generate_ack):     Only reached if setting_ack_handled=False (continue guard at line ~1396)
   
4. MUTUAL EXCLUSION BETWEEN WRITE POINTS
   - Points 2 & 3: Mutually exclusive via `setting_ack_handled` flag
     - Point 2 sets `setting_ack_handled = True` → Point 3 skipped
     - Point 3 only executes if `not setting_ack_handled`
   - Points 2/3 vs Point 4: `if setting_ack_handled: continue` skips Point 4
   - Point 1: Only for banned devices, `break` exits loop entirely
   
5. BUFFER DEDUP
   - MemoryHandler buffers are write-append only
   - RotatingFileHandler writes sequentially, no seek/rewrite
   - No retry logic on the forensic write path

=== Verification Method ===

To verify at runtime:
  1. Parse forensic_outbound.log as JSONL
  2. For each record, compute hash = sha256(conn_id + seq + raw_xml + ts_utc)
  3. Group by hash → assert all groups have count == 1
  4. Group by (conn_id, seq) → assert all groups have count == 1

The monotonic seq counter and mutual exclusion of write points guarantee
zero duplicate forensic writes by design.
