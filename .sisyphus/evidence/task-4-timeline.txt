## VERDICT for Hypothesis 3 (delivery timing):
- Timing is NOT the issue - polls happen every 40-80s on average
- Pending_frame would be delivered within 1-4 minutes max
- Unless something clears pending_frame before the next poll
- The ack_timeout_s in proxy config could be relevant - if timeout expires
  before next poll, ACK is never checked

## KEY CONCERN:
- ack_timeout_s needs to be LONGER than max poll interval (260s observed)
- If ack_timeout_s < 260s, some deliveries might time out before BOX confirms
